# CTF-Misc入门指北

## 一些名词解释

### 什么是CTF

**[CTF Capture The Flag]** 中文一般译作夺旗赛，在网络安全领域中指的是网络安全技术人员之间进行技术竞技的一种比赛形式。CTF 起源于 1996 年 DEFCON 全球黑客大会，以代替之前黑客们通过相发起真实攻击进行技术比拼的方式；其将安全相关的知识点抽象出来并加入到题目中，我们通过对知识点的理解认知，具体地进行实践来攻克题目。

### 什么是flag

主办方出题人通常会提供静态文件或是动态环境，选手通过进行攻防对抗、程序分析等形式，率先从主办方给出的比赛环境中得到一串具有一定格式的字符串或其他内容，并将其提交给主办方，从而夺得分数。为了方便称呼，我们把这样的内容称之为“flag”。通常flag满足`flag{xxxxxx}`或`比赛名称{xxxxxx}`诸如此类的格式（若有特殊格式会在题目中声明）。

### 什么是wp

wp全名为**Write Up，**是记录 CTF 比赛中解题过程的文章，通常包含题目描述、解题思路、解题过程、源码分析、脚本分析等内容。当然你也可以简单理解为解题报告——题解。

## Misc入门须知

**Misc 杂项**是 CTF 赛事的五大基础方向之一。安全杂项涉及到古典密码学、编码、隐写术、电子取证、数据分析等广度极高的安全手段及利用方法，选手需要确定手段或者方法，反向的去破译取证从而拿到 flag，Misc 是 CTF 比赛中广度最高的方向，需要各个方向都有涉猎。

Misc 手需要具备如下能力：

- 科学上网能力：科学上网是什么应该不用多说了吧，不懂的私底下问问就好

- 信息搜集能力：通常用于 OSINT（开源网络情报）类型的题目上，题目往往会提供一些关键信息，然后选手通过搜索引擎、社交平台等进行相关信息搜集，从各种公开的信息资源中寻找和获取有价值的情报。当然信息搜集能力不仅仅用于 OSINT 中，日常的学习过程以及一些比赛中，遇到冷门的知识点要懂得如何搜集相关文章以及解题方法

  （ps：OSINT学好了，你就是开盒高手（bushi））

- 自主学习能力： CTF 学习过程中，以自学刷题为主，前人引导为辅。在刷题过程中，利用前面提到的信息搜集能力，进行相关知识点的学习，并在事后及时做好知识点归纳总结，这样遇到相同考点的题目时就不会一头雾水。并且在一些赛事中，可能会遇到赛前从未见过的冷门考点，需要比赛过程中当场学习，所以自主学习能力是必不可少的

- 提问的能力：是的，如何提问也是一个非常重要的一项能力，详情请见这篇文章《提问的智慧》https://ctf.xidian.edu.cn/wiki/7

## Misc入门准备

Misc 手通常需要建立一个自己的工具箱，在学习过程中，不断地去丰富完善这个工具箱，而不是在解一道题目之前花费大量的时间在配置工具环境上。以下会给出一些 Misc 手使用频率较高的工具清单，部分工具具体配置方法请自行在网上查找：

1. 一门基础语言编译环境：这里推荐是 **Python** ，泛用性高上手简单（推荐anaconda或miniconda用于多版本部署和环境管理）
2. **CyberChef**：赛博厨师，必备的“网络瑞士军刀”，几乎全能的编解码工具

可以下载下来本地部署：https://github.com/gchq/CyberChef/releases/tag/v10.19.4

1. **010Editor/WinHex**：专业的文本编辑器和十六进制编辑器，可以较好的解析文件结构
2. **QR Research**：专业的二维码扫描识别软件，支持多个纠错等级
3. **StegSolve**：常用的图像分析工具，需要java环境
4. **ARCHPR**：压缩文件密码破解工具，支持多模式破解
5. **Audacity**：多轨音频处理工具
6. **WireShark**：流量分析取证工具
7. **exiftool**：文件元数据编辑工具
8. **Volatility**：内存分析取证工具
